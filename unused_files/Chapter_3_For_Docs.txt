บทที่ 3
การวิเคราะห์และออกแบบระบบ (System Analysis and Design)

ในบทนี้จะกล่าวถึงขั้นตอนการวิเคราะห์ความต้องการของระบบบริหารจัดการครุภัณฑ์ การออกแบบแผนภาพบริบท (Context Diagram) แผนภาพการไหลของข้อมูล (Data Flow Diagram) คำอธิบายการประมวลผล (Process Specification) และการออกแบบฐานข้อมูล (Database Design) เพื่อให้ระบบสามารถตอบสนองความต้องการของผู้ใช้งานได้อย่างมีประสิทธิภาพ

3.1 แผนภาพบริบท (Context Diagram)
Context Diagram แสดงภาพรวมความสัมพันธ์ระหว่างระบบกับหน่วยงานภายนอก (External Entities) ทั้ง 3 กลุ่มหลัก คือ ผู้ดูแลระบบ (Admin), เจ้าหน้าที่ (Inspector) และผู้บริหาร (Viewer)

3.2 แผนภาพการไหลของข้อมูลระดับที่ 0 (DFD Level 0)
กระบวนการทำงานหลักในภาพรวมประกอบด้วย:
1.0 จัดการข้อมูลผู้ใช้งาน
2.0 จัดการข้อมูลพื้นฐาน (หน่วยงาน/สถานที่)
3.0 จัดการข้อมูลครุภัณฑ์
4.0 การตรวจสอบและสแกนสภาพครุภัณฑ์
5.0 จัดการยืม-คืนครุภัณฑ์
6.0 ระบบออกรายงานและประวัติการใช้งาน

3.4 คำอธิบายการไหลของข้อมูล (ตัวอย่าง)
- ข้อมูลครุภัณฑ์: ประกอบด้วย รหัสครุภัณฑ์, ชื่อ, ซีเรียลนัมเบอร์, สถานะ และรูปภาพ
- ข้อมูลการยืม-คืน: ประกอบด้วย รายละเอียดการยืม, วันที่ยืม, ผู้อนุมัติ และวันที่คืน
- ข้อมูลการตรวจสภาพ: ประกอบด้วย ผลการสแกนจากโทรศัพท์มือถือ, วันเวลา และสถานะสภาพปกติ/ชำรุด

3.5 แหล่งเก็บข้อมูลหลัก (Data Stores)
D1 ข้อมูลผู้ใช้งาน: เก็บข้อมูลบัญชีและสิทธิ์
D2 ข้อมูลหน่วยงาน: เก็บรายชื่อคณะ/แผนก
D3 ข้อมูลสถานที่: เก็บข้อมูลอาคารและห้อง
D4 ข้อมูลครุภัณฑ์: แฟ้มข้อมูลหลักของสินทรัพย์
D5 ข้อมูลการตรวจสภาพ: เก็บประวัติการสแกนสำรวจ
D8 ข้อมูลการยืมคืน: เก็บประวัติการยืมและคืน

3.7 พจนานุกรมข้อมูล (Data Dictionary)
ตาราง Users: ประกอบด้วย user_id, username, password, fullname, role
ตาราง Assets: ประกอบด้วย asset_id, asset_name, serial_number, quantity, status, location_id
ตาราง Asset_Check: ประกอบด้วย check_id, asset_id, user_id, check_date, check_status

3.8 ความสัมพันธ์ของข้อมูล (Entity Relationship)
- ผู้ดูแลระบบ (Users) จัดการข้อมูลครุภัณฑ์ (Assets)
- ครุภัณฑ์ (Assets) สังกัดอยู่ตามหน่วยงาน (Departments) และสถานที่ (Locations)
- เจ้าหน้าที่ (Users) ทำการตรวจสภาพ (Asset_Check) และบันทึกประวัติการยืม (Borrow)

(หมายเหตุ: สำหรับรูปภาพแผนภาพ Mermaid และ DFD กรุณาใช้ไฟล์รูปภาพที่บันทึกจากระบบต้นฉบับมาแนบประกอบในเล่มรายงาน)
